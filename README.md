## 📋 Overview

**Abogato** is a web platform that allows clients to manage their legal procedures digitally — without visiting a physical office. The system connects clients with lawyers through a structured flow of tickets and legal documents automatically generated from templates.

Key goals:
- Allow clients to initiate legal procedures online
- Automate legal document generation via dynamic templates
- Facilitate document review and approval by lawyers
- Provide a ticket system to track each procedure's status
- Guarantee document generation in under 15 minutes
- Provide an AI-powered legal assistant for quick consultations

## ✨ Features

### 👤 Client
- Create and manage legal tickets with title, description, and priority
- Attach files (PDF, JPG, PNG up to 10MB) to tickets
- Select a document template and fill out a guided form with dynamic fields
- Preview the generated document before submission
- Send the document to the assigned lawyer for review
- Download approved documents
- Request ticket reopening if needed
- Register and manage real estate assets (bienes)
- Consult the AI legal assistant for quick answers
- View suggested FAQ questions in the chat interface

### 🧑‍⚖️ Lawyer
- View all assigned tickets and available cases
- Take a case and advance its status (open → in progress → resolved → closed)
- Review, approve, or reject client-generated documents
- Add internal comments (not visible to the client)
- Approve or reject reopening requests

### 🛠️ Admin
- Manage all system users
- Create and configure legal document templates with dynamic fields (text, date, number)
- Activate or deactivate templates
- View all tickets across the system
- Manage the AI assistant's FAQ knowledge base (add, edit, activate/deactivate, delete)
- View all questions submitted to the AI assistant
- Monitor AI assistant performance metrics (resolution rate, escalations)

### 🤖 AI Legal Assistant
- Floating chat widget available on all pages
- Suggests example questions on first open (HU-044)
- Answers frequently asked questions from a managed FAQ database (HU-045)
- Falls back to Groq (LLaMA 3.1) for questions not in FAQ
- Automatically escalates unresolved queries to a lawyer ticket (HU-045)
- Collects user feedback on responses (👍 / 👎)
- Saves all conversations to Supabase for admin review

### 📰 Blog
- Legal articles covering common Costa Rican procedures
- Topics include: divorce, vehicle transfer, property transfer, civil marriage, digital signatures, and more

| Status | Description |
|--------|-------------|
| `draft` | Draft, not yet submitted |

| `approved` | Approved by the lawyer |
| `rejected` | Rejected by the lawyer |

## 🔐 Security

- **Supabase Auth** — session-based authentication with cookies
- **Row Level Security (RLS)** — data isolation per user role at the database level
- **Route middleware** — `auth.ts` guards all protected routes; `lawyer.ts` restricts lawyer/admin areas
- **SSR disabled** — `ssr: false` in `nuxt.config.ts` for full Supabase Auth compatibility on the client
- **Inactivity logout** — automatic session termination after inactivity (`useInactivityLogout`)
- **Service Role Key** — server-only Supabase key for admin APIs, never exposed to the browser

## 📦 Installation

```bash
# 1. Clone the repository
git clone https://github.com/LuchoTechCR/abogato-server.git
cd abogato-server/abogato-webapp

# 2. Install dependencies
pnpm install

# 3. Set up environment variables
# Create a .env file in the root of abogato-webapp:
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=sb_publishable_your_key_here

# Server-only (required for admin APIs + AI FAQ sync)
# Use the Supabase "service_role" key (sb_secret_...) — NEVER expose it to the browser.
SUPABASE_SERVICE_ROLE_KEY=sb_secret_your_service_role_key_here

# Groq API key for the AI legal assistant
GROQ_API_KEY=your_groq_api_key_here

# 4. Start the development server
pnpm dev
# App available at http://localhost:3000
```

**Requirements:** Node.js v20+, pnpm

## 🤖 AI Assistant Architecture

The AI legal assistant follows a two-step resolution strategy:

```
User message
     │
     ▼
Search FAQ table (Supabase)
     │
     ├── Match found ──► Return FAQ answer instantly
     │
     └── No match ────► Call Groq API (LLaMA 3.1 8B)
                              │
                              ├── Confident answer ──► Return response
                              │
                              └── Uncertain ──────────► Suggest creating a ticket
```

All conversations are stored in `ai_conversations` with:
- `was_escalated` — whether the query was escalated to a lawyer
- `feedback` — user thumbs up/down rating
- `session_id` — groups messages per conversation session
